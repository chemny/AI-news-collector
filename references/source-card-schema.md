# Source Card Schema

The normalized source card is the stable contract between collection and later stages such as topic mining, writing, platform adaptation, and review.

## Required Fields

| Field | Type | Meaning |
|---|---|---|
| `item_uid` | string | Stable ID generated by this collector. |
| `title` | string | Original or lightly cleaned title. |
| `title_zh` | string/null | Chinese title for reports and downstream topic mining. |
| `title_source` | string | `raw`, `cleaned`, `generated`, or `missing`. X/Twitter items without titles may use generated titles. |
| `title_quality_flags_json` | JSON array string | Title quality flags, such as `missing_title`, `generic_title`, `too_long`, or `contains_url`. |
| `normalized_title` | string | Lowercased, punctuation-light title for dedupe. |
| `source_name` | string | Human-readable source, such as `Claude Blog`, `Hacker News`, or the actual media/account name returned by an aggregator. |
| `source_type` | string | `x`, `blog`, `rss`, `newsletter`, `podcast`, `video`, `paper`, `news`, `website`, `github`, or `other`. |
| `platform` | string | `x`, `youtube`, `web`, `rss`, `github`, `wechat`, `xiaohongshu`, `bilibili`, `other`. |
| `url` | string | Original URL. |
| `canonical_url` | string | URL after tracking parameters and fragments are removed. |
| `author` | string/null | Author, builder, publication, account, or organization. |
| `author_handle` | string/null | Platform handle, such as an X username. |
| `published_at` | string/null | Source publish time in ISO 8601 when available. |
| `collected_at` | string | Collector run timestamp in ISO 8601. |
| `updated_at` | string | Last local update timestamp in ISO 8601. |
| `summary` | string | Standardized summary. Use source summary only if useful; otherwise generate from available content. |
| `summary_zh` | string/null | Chinese summary for reports and downstream topic mining. |
| `summary_source` | string | `raw`, `cleaned`, `generated`, or `missing`. |
| `summary_quality_flags_json` | JSON array string | Summary quality flags, such as `missing_summary`, `too_short`, `contains_url`, or `noise_summary`. |
| `content` | string/null | Full text, transcript, or long excerpt when available. |
| `category` | string/null | Source category such as `ai-models`, `ai-products`, `industry`, `paper`, `tip`, `builder-update`. |
| `tags_json` | JSON array string | Tags or keywords. |
| `language` | string | `zh`, `en`, `mixed`, or `unknown`. |
| `heat` | string/null | Heat signal, such as stars, points, likes, or source ranking. |
| `like_count` | integer/null | Normalized likes, stars, upvotes, or equivalent positive reactions. |
| `comment_count` | integer/null | Normalized comments or discussion count. |
| `repost_count` | integer/null | Normalized shares, reposts, retweets, forks, or equivalent spread signal. |
| `favorite_count` | integer/null | Normalized saves, bookmarks, favorites, or collections. |
| `view_count` | integer/null | Normalized views, impressions, or plays. |
| `quote_count` | integer/null | Quote repost count where the platform exposes it. |
| `reply_count` | integer/null | Reply count where separate from comments. |
| `engagement_score` | number | Cross-platform baseline heat score calculated from available metrics. Missing metrics contribute zero rather than blocking the item. |
| `platform_engagement_percentile` | number/null | Engagement percentile within the same platform and daily bucket. If the platform has no available metrics, use `0.5` when calculating collector ranking. |
| `engagement_velocity_score` | number | Engagement score divided by item age in hours. Missing metrics contribute zero. |
| `freshness_score` | number | 24-hour freshness score from `1.0` for just-published to `0.0` at the window boundary. |
| `collector_rank_score` | number | Collector-level baseline rank score combining platform percentile, velocity, source priority, reliability, and freshness. This is not the final topic score. |
| `engagement_raw_json` | JSON object string | Raw platform metrics so no source-specific signal is lost. |
| `source_tags_json` | JSON array string | Tags/categories from the original source, such as GitHub topics or RSS categories. |
| `entity_tags_json` | JSON array string | Detected companies, products, people, and named entities. |
| `topic_tags_json` | JSON array string | Detected content themes, such as `AI 编程`, `Agent`, `模型发布`, or `开源项目`. |
| `audience_tags_json` | JSON array string | Likely audience groups, such as `程序员`, `产品经理`, `创作者`, `创业者`, or `研究者`. |
| `platform_hint_json` | JSON array string | Initial downstream platform hints such as `wechat`, `xiaohongshu`, `douyin`, `bilibili`, or `x`. |
| `tag_confidence_json` | JSON object string | Confidence values for rule/AI-generated tags. |
| `source_reliability` | number | Baseline credibility score, defaulting to `0.5` when unknown. |
| `source_priority` | number | Local source weight used later by ranking, defaulting to `1`. |
| `author_followers` | integer/null | Author/account follower count if available. |
| `author_verified` | integer | `1` if the source reports verified/official author status, otherwise `0`. |
| `author_profile_json` | JSON object string | Raw author profile data. |
| `is_official_source` | integer | `1` for detected official company/product/publication source. |
| `is_primary_source` | integer | `1` for likely first-hand source, including official blogs, papers, repos, and creator-owned posts. |
| `content_type` | string/null | `news`, `launch`, `tutorial`, `paper`, `repo`, `discussion`, or source-provided type. |
| `media_urls_json` | JSON array string | Image, video, thumbnail, or cover URLs. |
| `related_urls_json` | JSON array string | Related links surfaced by the source item. |
| `raw_metrics_at` | string/null | Timestamp when dynamic engagement metrics were collected. |
| `provider` | string/null | Adapter provider path, such as `nitter` for X builder monitoring. |
| `is_repost` | integer | `1` when the X item is a repost/retweet surfaced by the tracked builder. |
| `original_author` | string/null | Original author name if known for reposted content. |
| `original_handle` | string/null | Original author handle if known for reposted content. |
| `raw_record_uid` | string/null | Link back to `raw_source_records.raw_uid`. Use this to audit original external data. |
| `raw_source` | string | Adapter name, such as `aihot` or `follow_builders`. |
| `dedupe_key` | string | Primary dedupe key. Usually URL hash when URL exists. |
| `content_hash` | string | Hash of title + summary/content excerpt. |
| `daily_bucket` | string | Local date bucket, `YYYY-MM-DD`. |
| `status` | string | `new`, `duplicate_today`, `duplicate_recent`, `skipped`, or `archived`. |
| `translation_status` | string | `source_zh`, `translated`, `needs_translation`, or `not_needed`. |
| `mapping_status` | string | `ok`, `partial`, or `failed`, indicating whether source fields were mapped cleanly. |
| `missing_fields_json` | JSON array string | Required or important normalized fields missing after mapping. |
| `quality_flags_json` | JSON array string | Processing-layer quality flags such as `missing_summary`, `missing_content`, or `needs_translation`. |
| `processing_version` | string | Normalization contract/rule version, used for future reprocessing audits. |
| `ai_processing_model` | string/null | Model or local processor used for generated title/summary. |
| `ai_processing_version` | string/null | Title/summary generation rule version. |

## Status Semantics

`new`: eligible for today's daily briefing and topic mining.

`duplicate_today`: same or highly similar item already exists in the same daily bucket.

`duplicate_recent`: same or highly similar item exists in the previous recent-day dedupe window.

`skipped`: missing required fields, outside time window, unsupported source, or otherwise not usable.

`archived`: retained but not active for current workflows.

## Chinese Field Rules

Reports and topic-mining skills should read `title_zh` and `summary_zh` first.

If `translation_status = 'needs_translation'`, the item can be stored but should be translated before entering user-facing reports.
